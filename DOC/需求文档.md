## 1. 项目背景

### 1.1 业务背景

公司主营跨境电商综合服务，业务覆盖以下场景：

- **店铺购买**：客户需要购买海外电商平台（如 Amazon、Shopee、Lazada 等）的成熟店铺；
- **开店培训**：帮助国内工厂/贸易商开设海外店铺，提供全流程培训指导；
- **合作工厂**：与国内工厂对接，获取一手货源，联合运营海外市场；
- **物流合作**：与海外仓、跨境物流公司建立长期合作关系。

随着业务扩张，客户数量快速增长，原先使用 Excel 表格管理客户信息的方式已无法满足需求：

- 多人协作困难，信息无法实时同步；
- 数据散落，无法快速筛选和统计；
- 缺乏规范的录入流程，客户信息质量参差不齐；
- 无法安全地对外共享客户资料供合作方浏览。

因此，需要开发一套**轻量级企业客户资料管理系统（CRM）**，实现客户信息的规范化录入、高效查询、数据导出和安全共享。

### 1.2 目标用户

| 角色 | 说明 | 核心需求 |
|------|------|---------|
| 业务员 | 一线销售人员，负责客户开发 | 录入客户信息、查看自己跟进中的客户 |
| 销售主管 | 团队管理者 | 查看团队客户全貌、导出Excel报表 |
| 合作方 | 外部合作伙伴（工厂、物流等） | 通过共享链接浏览已签约客户基本资料 |

### 2.2 功能详细描述

#### F1 — 客户信息录入

**优先级**：P0（核心功能）

**描述**：提供标准化的客户信息录入表单，支持必填校验和格式验证。

**录入字段**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 公司名称 | ✅ | 客户公司或店铺名称，最长200字符 |
| 联系人 | ✅ | 主要对接人姓名，最长100字符 |
| 联系电话 | ❌ | 手机号码或座机（含国际区号） |
| 邮箱 | ❌ | 电子邮箱地址，须校验格式 |
| 微信号 | ❌ | 微信账号 |
| 国家/地区 | ❌ | 客户所在国家或地区 |
| 详细地址 | ❌ | 公司或仓库的物理地址 |
| 网站 | ❌ | 公司官网或店铺链接 |
| 客户类型 | ✅ | 下拉选择：店铺购买/开店培训/合作工厂/物流合作/其他 |
| 服务需求 | ❌ | 文本描述，记录客户的具体需求 |
| 客户来源 | ❌ | 获客渠道：展会/官网/朋友推荐/LinkedIn等 |
| 跟进状态 | ❌ | 下拉选择，默认"潜在客户" |
| 备注 | ❌ | 补充信息 |
| 录入人 | ❌ | 谁录入的这条客户信息 |

**验收标准**：
- [ ] 必填字段为空时阻止提交并提示
- [ ] 邮箱格式不正确时有校验提示
- [ ] 客户类型、跟进状态为下拉选择，不可随意输入
- [ ] 新增成功后自动跳转到列表页并刷新

#### F2 — 客户列表查询

**优先级**：P0

**描述**：分页展示客户列表，支持多条件组合筛选。

**筛选条件**：
- 关键词搜索（模糊匹配公司名和联系人）
- 客户类型（精确匹配）
- 跟进状态（精确匹配）
- 国家/地区（精确匹配）

**列表展示列**：
序号、公司名称、联系人、电话、国家/地区、客户类型、跟进状态、录入人、创建时间、操作按钮

**验收标准**：
- [ ] 默认按创建时间倒序排列
- [ ] 支持 10/20/50 条每页切换
- [ ] 筛选条件可组合使用
- [ ] 关键词搜索支持中文和英文
- [ ] 点击公司名称可查看详情

#### F3 — 客户详情查看

**优先级**：P1

**描述**：以模态框形式展示客户全部字段信息。

**验收标准**：
- [ ] 显示客户的所有字段（包括备注和服务需求）
- [ ] 网站字段可点击跳转
- [ ] 布局清晰，按基本信息/业务信息/地区信息分组

#### F4 — 客户信息编辑

**优先级**：P1

**描述**：编辑已有客户的任意字段。

**验收标准**：
- [ ] 编辑表单预填充当前数据
- [ ] 编辑后更新时间自动刷新
- [ ] 编辑成功后留在当前分页

#### F5 — 客户删除

**优先级**：P1

**描述**：删除客户信息（逻辑删除，数据不物理删除）。

**验收标准**：
- [ ] 删除前弹出确认对话框
- [ ] 删除后列表自动刷新
- [ ] 被删除的记录不会出现在列表中

#### F6 — Excel 导出

**优先级**：P0

**描述**：将客户数据导出为 Excel 文件（.xlsx 格式）。

**导出内容**：全部字段（17列），含表头样式美化。

**验收标准**：
- [ ] 支持全量导出和根据当前筛选条件导出
- [ ] 表头有加粗、蓝色背景样式
- [ ] 日期格式为 yyyy-MM-dd HH:mm:ss
- [ ] 文件名包含"客户资料"关键词
- [ ] 浏览器自动触发文件下载

#### F7 — 共享浏览

**优先级**：P2

**描述**：生成独立的 Web 页面，供外部合作方浏览已签约客户的基本信息。

**安全限制**：
- 只展示状态为"已签约"的客户
- 不显示手机号、邮箱、微信号、详细地址等私密信息
- 只读页面，无编辑/删除操作
- 提供搜索和统计概览

**验收标准**：
- [ ] 独立页面，可通过链接直接访问
- [ ] 仅展示已签约客户
- [ ] 敏感字段被隐藏
- [ ] 有搜索功能和简单统计

## 3. 非功能性需求

### 3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 列表查询响应时间 | ≤ 500ms（1000条数据内） |
| Excel 导出时间 | ≤ 5s（10000条数据内） |
| 并发用户数 | 支持 50 并发 |
| 页面首屏加载 | ≤ 2s |

### 3.2 安全性要求

- 后端参数校验：所有入参需经 `@Valid` 校验；
- SQL 注入防护：使用 MyBatis-Plus 参数化查询，禁止拼接 SQL；
- XSS 防护：前端输出时对 HTML 特殊字符转义；
- CORS 控制：生产环境限制允许跨域的域名白名单；
- 逻辑删除：数据不物理删除，可追溯、可恢复。

### 3.3 兼容性要求

- 浏览器：Chrome 90+、Firefox 90+、Edge 90+、Safari 14+
- 数据库：MySQL 8.0+
- Java：JDK 8/11/17
- 操作系统：Windows / Linux / macOS

### 3.4 可用性要求

- 界面简洁直观，业务员无需培训即可使用；
- 操作有明确的成功/失败反馈；
- 表单校验在提交时触发，即时提示错误；
- 列表为空时有友好的空状态提示。

## 4. 数据字典

### 4.1 客户类型枚举

| 值 | 说明 |
|----|------|
| 店铺购买 | 客户需要购买海外电商平台店铺 |
| 开店培训 | 客户需要开店全流程培训指导 |
| 合作工厂 | 工厂方提供货源，联合运营 |
| 物流合作 | 海外仓、跨境物流合作方 |
| 其他 | 未分类的客户类型 |

### 4.2 跟进状态枚举

| 值 | 说明 |
|----|------|
| 潜在客户 | 刚录入，尚未开始跟进 |
| 初步接洽 | 已进行首次沟通 |
| 意向明确 | 客户表达了明确的合作意向 |
| 已签约 | 已完成签约或购买 |
| 已流失 | 客户明确拒绝或长期无响应 |

## 6. 术语表

| 术语 | 说明 |
|------|------|
| CRM | Customer Relationship Management，客户关系管理 |
| 逻辑删除 | 数据库标记删除（is_deleted=1），不物理删除数据 |
| 脱敏 | 隐藏或模糊处理敏感字段（手机号、邮箱等） |
