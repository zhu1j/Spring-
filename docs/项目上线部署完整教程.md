# 项目上线部署完整教程

> 适用项目：CRM 客户资料管理系统（Spring Boot + 静态前端）
> 目标环境：Linux 服务器（CentOS 7 / Ubuntu 20.04+）

---

## 一、部署架构概览

```
用户浏览器
    │
    ▼
┌─────────────────────┐
│   Nginx (端口 80)    │  ← 反向代理 + 托管前端静态文件
│   /              → 前端页面 (index.html, js, css)
│   /api/*        → 转发到后端 8080
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│  Spring Boot JAR    │  ← 后端服务 (端口 8080)
│  java -jar crm.jar  │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│  MySQL 8.0          │  ← 数据库 (端口 3306)
│  数据库：crm_db      │
└─────────────────────┘
```

## 三、服务器环境搭建

### 3.1 安装 JDK 8

**CentOS：**
```bash
# 安装
sudo yum install -y java-1.8.0-openjdk java-1.8.0-openjdk-devel

# 验证
java -version
# 输出: openjdk version "1.8.0_xxx"
```

**Ubuntu：**
```bash
sudo apt update
sudo apt install -y openjdk-8-jdk
java -version
```

**设置环境变量**（可选，一般 yum/apt 安装后自动配置）：
```bash
# 编辑 /etc/profile 追加
export JAVA_HOME=/usr/lib/jvm/java-1.8.0-openjdk
export PATH=$JAVA_HOME/bin:$PATH

# 生效
source /etc/profile
```

---

### 3.2 安装 MySQL 8.0

**CentOS 7：**
```bash
# 添加 MySQL 官方仓库
sudo yum install -y https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm

# 安装
sudo yum install -y mysql-community-server

# 启动
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码
sudo grep 'temporary password' /var/log/mysqld.log
```

**Ubuntu 20.04：**
```bash
sudo apt update
sudo apt install -y mysql-server

# 启动
sudo systemctl start mysql
sudo systemctl enable mysql
```

**初始化数据库：**
```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE crm_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 创建用户并授权（替换 your_password 为实际密码）
CREATE USER 'crm_user'@'localhost' IDENTIFIED BY 'your_strong_password';
GRANT ALL PRIVILEGES ON crm_db.* TO 'crm_user'@'localhost';
FLUSH PRIVILEGES;

-- 导入表结构（从本地导出后上传到服务器执行）
USE crm_db;
SOURCE /opt/crm/schema.sql;
```

**导出本地表结构：**
```bash
# 在本地开发机执行，导出表结构
mysqldump -u root -p --no-data crm_db > schema.sql

# 把 schema.sql 上传到服务器
scp schema.sql root@你的服务器IP:/opt/crm/
```

## 四、配置生产环境的 `application.yml`

在 `src/main/resources/` 下新建 `application-prod.yml`（生产环境专用配置）：

```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/crm_db?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai&useSSL=true
    username: crm_user                # 上面创建的用户
    password: your_strong_password    # 上面设置的密码
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      idle-timeout: 300000
      max-lifetime: 600000
      connection-timeout: 30000

  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: GMT+8
    default-property-inclusion: non_null

mybatis-plus:
  type-aliases-package: com.crm.entity
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.nologging.NoLoggingImpl  # 生产关闭 SQL 日志
  global-config:
    db-config:
      logic-delete-field: isDeleted
      logic-delete-value: 1
      logic-not-delete-value: 0
      id-type: auto

logging:
  level:
    com.crm: info                    # 生产用 info 级别
    org.springframework: warn
  file:
    path: /opt/crm/logs              # 日志文件路径
    name: /opt/crm/logs/crm.log
```

**关键生产配置变化：**

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| MySQL 密码 | `root/root` | 强密码 |
| SQL 日志 | `StdOutImpl`（控制台输出） | `NoLoggingImpl`（关闭） |
| 日志级别 | `debug` | `info` / `warn` |
| 日志输出 | 控制台 | 文件 `/opt/crm/logs/` |
| `useSSL` | `false` | `true` |
| CORS | `allowedOriginPattern("*")` | 改为具体域名 |

## 五、打包与上传

### 5.1 修改前端 API 地址

部署前，确认 `front/js/api.js` 里的 `BASE_URL` 正确：

```javascript
// 本地开发
const BASE_URL = 'http://localhost:8080';

// 服务器部署（如果 Nginx 做了反向代理，可以改成空字符串用相对路径）
const BASE_URL = '';   // 前端和后端同域名同端口时用这个

// 或者
const BASE_URL = 'https://你的域名.com';
```

### 5.2 打包后端

```bash
# 在项目根目录（pom.xml 所在目录）执行
mvn clean package -DskipTests
```

打包成功后，JAR 文件在 `target/crm-backend-1.0.0.jar`。

### 5.3 上传到服务器

```bash
# 在本地执行，上传文件
# 1. 上传 JAR 包
scp target/crm-backend-1.0.0.jar root@你的服务器IP:/opt/crm/

# 2. 上传前端文件
scp -r front/* root@你的服务器IP:/opt/crm/front/

# 3. 上传 schema.sql
scp schema.sql root@你的服务器IP:/opt/crm/
```

### 5.4 创建服务器目录结构

```bash
# SSH 登录服务器后
ssh root@你的服务器IP

mkdir -p /opt/crm/logs
mkdir -p /opt/crm/front
```

## 六、配置 Nginx

### 6.1 编辑 Nginx 配置

```bash
sudo vim /etc/nginx/conf.d/crm.conf
```

写入：

```nginx
server {
    listen 80;
    server_name 你的域名.com;   # 有域名填域名，没有填服务器IP

    # 前端静态文件
    location / {
        root /opt/crm/front;
        index index.html;
        try_files $uri $uri/ /index.html;   # SPA 路由回退
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时时间
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }

    # gzip 压缩（加快前端加载）
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        root /opt/crm/front;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

验证并重载：

```bash
# 检查配置文件语法
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

## 七、启动后端服务

### 7.1 手动启动（临时测试）

```bash
cd /opt/crm

# 指定生产配置启动
java -jar crm-backend-1.0.0.jar --spring.profiles.active=prod

# 看到 "客户资料管理系统启动成功！" 即成功
# 按 Ctrl+C 停止
```

### 7.2 配置为系统服务（推荐，后台运行 + 开机自启）

创建 systemd 服务文件：

```bash
sudo vim /etc/systemd/system/crm.service
```

写入：

```ini
[Unit]
Description=CRM Backend Service
After=network.target mysql.service
Wants=mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/crm
ExecStart=/usr/bin/java -jar -Xms256m -Xmx512m /opt/crm/crm-backend-1.0.0.jar --spring.profiles.active=prod
ExecStop=/bin/kill -15 $MAINPID
Restart=on-failure
RestartSec=10

# 日志
StandardOutput=append:/opt/crm/logs/stdout.log
StandardError=append:/opt/crm/logs/stderr.log

[Install]
WantedBy=multi-user.target
```

**JVM 参数说明：**

| 参数 | 含义 | 建议值 |
|------|------|--------|
| `-Xms256m` | 初始堆内存 | 256MB |
| `-Xmx512m` | 最大堆内存 | 512MB（按服务器内存调整） |

**启动服务：**
```bash
sudo systemctl daemon-reload        # 重新加载 systemd 配置
sudo systemctl start crm            # 启动
sudo systemctl status crm           # 查看状态
sudo systemctl enable crm           # 开机自启
```

**常用运维命令：**
```bash
systemctl status crm        # 查看运行状态
systemctl restart crm       # 重启
systemctl stop crm          # 停止
journalctl -u crm -f        # 实时查看日志
journalctl -u crm -n 100    # 查看最近 100 行日志
```

## 八、防火墙与安全组

### 8.1 服务器防火墙

```bash
# 开放 HTTP (80) 端口
sudo firewall-cmd --add-service=http --permanent

# 如果 SSH 是 22，也确保开放
sudo firewall-cmd --add-service=ssh --permanent

sudo firewall-cmd --reload

# 查看已开放端口
sudo firewall-cmd --list-all
```

### 8.2 云服务器安全组

如果你用的是阿里云/腾讯云/AWS 等云服务器，还需要在**安全组/防火墙规则**中添加：

| 端口 | 协议 | 来源 | 用途 |
|------|------|------|------|
| 22 | TCP | 你的IP | SSH 远程管理 |
| 80 | TCP | 0.0.0.0/0 | HTTP 访问 |
| 443 | TCP | 0.0.0.0/0 | HTTPS（配了 SSL 后） |

> ⚠️ **不要开放 8080 和 3306 端口**！后端和数据库只监听本地 127.0.0.1，外部通过 Nginx 的 80 端口访问即可。

## 九、配置 HTTPS（SSL 证书）

### 9.1 使用 Let's Encrypt（免费）

```bash
# 安装 certbot
# CentOS：
sudo yum install -y certbot python3-certbot-nginx

# Ubuntu：
sudo apt install -y certbot python3-certbot-nginx

# 自动配置 SSL
sudo certbot --nginx -d 你的域名.com

# 证书有效期 90 天，设置自动续期
sudo certbot renew --dry-run   # 测试自动续期
# certbot 默认添加了定时任务，无需手动配置
```

### 9.2 配置 HTTP 自动跳转 HTTPS

certbot 会自动加，如果没有，在 Nginx 的 `crm.conf` 最前面加：

```nginx
server {
    listen 80;
    server_name 你的域名.com;
    return 301 https://$host$request_uri;
}
```

## 十、生产环境检查清单

| # | 检查项 | 命令 / 操作 |
|---|--------|------------|
| 1 | MySQL 密码是不是弱密码 | `SELECT user,host FROM mysql.user;` |
| 2 | `application-prod.yml` 里日志是否为 info/warn | 检查 `logging.level` |
| 3 | SQL 日志是否关闭 | `log-impl: NoLoggingImpl` |
| 4 | CORS 是否限制为具体域名 | `addAllowedOrigin("https://你的域名")` |
| 5 | 8080 端口是否只监听本地 | `netstat -tlnp | grep 8080`（应显示 127.0.0.1） |
| 6 | Nginx 配置是否正确 | `nginx -t` |
| 7 | 服务是否设为开机自启 | `systemctl is-enabled crm` |
| 8 | 防火墙是否只开 80/443 | `firewall-cmd --list-all` |
| 9 | 浏览器访问能否打开 | 访问 `http://你的服务器IP` |

## 十二、常见线上问题排查

| 问题 | 排查命令 |
|------|---------|
| 网站打不开 | `systemctl status nginx` / `systemctl status crm` |
| 502 Bad Gateway | 后端没启动，检查 `systemctl status crm` |
| 数据库连接失败 | `systemctl status mysql` / 检查 `application-prod.yml` 密码 |
| 内存溢出 | `journalctl -u crm | grep OutOfMemory` |
| 端口被占用 | `netstat -tlnp | grep 端口号` |
| 查看后端日志 | `journalctl -u crm -n 50` 或 `tail -f /opt/crm/logs/crm.log` |
